# zvec — векторизация знаний для LLM

zvec превращает документы проекта (регламенты, описания процессов, документацию) в **векторный индекс** и ищет по ним **по смыслу**, а не по словам. Это даёт LLM-ассистентам (Cursor, Claude Code, opencode) быстрый доступ к нужному контексту: вы спрашиваете «как учитываются товары на складе» — и получаете релевантные фрагменты, даже если слова не совпали.

**Как это работает:** текст разбивается на куски, каждый кусок нейросеть `multilingual-e5-small` превращает в вектор (эмбеддинг) и кладёт в локальную векторную БД [ZVec](https://www.npmjs.com/package/@zvec/zvec). При поиске ваш запрос тоже векторизуется — и zvec находит ближайшие по смыслу фрагменты.

- **Локально и приватно:** модель работает на вашей машине, без внешних API.
- **Мультиязычно:** русский и английский в одном индексе.
- **Универсально:** индексируете любую папку, какую укажете.

---

## Надёжность: при сбое ничего не ломается

zvec спроектирован так, чтобы индексация никогда не повредила рабочую базу, а поиск оставался доступен. Каждая индексация пишется в отдельную **staging**-копию, пока активная база продолжает отвечать на запросы. Только после успешной проверки staging **атомарно** становится активной — через замену одного файла-манифеста. Если процесс прервался, на диске остаётся валидная предыдущая версия, и следующий поиск работает с ней. Если активная версия всё же повредилась, zvec автоматически откатывается к предыдущей. Файл базы не удерживается открытым между операциями, поэтому одновременные поиск и индексация не конфликтуют и не блокируют друг друга.

## Требования

**Node.js ≥ 18**, **Python ≥ 3.9**, `pip install sentence-transformers`.
При первом запуске модель скачивается с HuggingFace (~120 МБ) и кешируется.

```bash
cd zvec-tool
npm install
```

---

## Быстрый старт

```bash
# Путь к папке с документами — обязателен (любая директория на ваш выбор)
node zvec.mjs index ./docs
node zvec.mjs search "учёт товаров" --top 8
node zvec.mjs status
```

---

## Команды

`node zvec.mjs <command>`

| Команда | Описание |
|---------|----------|
| `index <путь>` | Переиндексация указанного каталога (staging → атомарный promote) |
| `search <запрос>` | Семантический поиск |
| `status` | Статус индекса |
| `doctor` | Диагностика (probe DB + кто держит lock) |
| `versions` | Список версий БД |
| `rollback <ver-id>` | Откат на версию |
| `backups` / `backup` | Список / создание резервной копии |

**Поиск:** `--top N` (по умолчанию 8), `--json`. Результат: путь к файлу, `score` (косинусное сходство, `> 0.7` — сильное совпадение), превью 220 символов.

> Путь индексации можно передать позиционно (`index ./docs`), флагом (`index --root ./docs`) или через переменную `ZVEC_INDEX_ROOT`.

## Переменные окружения

Основные (дефолты — в `lib/constants.mjs`, переопределения — в `lib/config.mjs`):

- `ZVEC_INDEX_ROOT` — каталог для индексации (**обязателен**, либо передайте путь в `index`)
- `PROJECT_ROOT` — корень проекта (по умолчанию родитель `zvec-tool/`)
- `ZVEC_EMBEDDING_MODEL` — модель (по умолчанию `intfloat/multilingual-e5-small`)
- `ZVEC_EXTENSIONS` — расширения файлов (по умолчанию `[".md"]`)
- `ZVEC_IGNORE_DIRS` / `ZVEC_EXCLUDE_FILES` — что пропускать
- `ZVEC_CHUNK_SIZE` / `ZVEC_CHUNK_OVERLAP` — чанкинг (1600 / 0)
- `ZVEC_PYTHON` — Python-исполняемый файл

Таймауты и префиксы эмбеддингов (`ZVEC_EMBED_TIMEOUT_MS`, `ZVEC_EMBED_STARTUP_MS`, `ZVEC_EMBED_QUERY_PREFIX`, `ZVEC_EMBED_DOC_PREFIX`, `ZVEC_LOCK_WAIT_MS` и др.) — см. `lib/config.mjs`.

## Диагностика

```bash
node zvec.mjs doctor     # открывает БД, показывает docCount и держателя lock
```

| Симптом | Решение |
|---------|---------|
| `ZVEC_BUSY` (exit 2) | Другой процесс держит lock — подождать или завершить |
| `docCount: 0` | Индекс пуст: `node zvec.mjs index <путь>` |
| `Embed worker not found` | Проверить наличие `embed_worker.py` |
| `sentence-transformers not found` | `pip install sentence-transformers` |
| Lock не сбрасывается | Удалить `.zvec/session.lock` вручную |

## Cursor и AI-агенты

- **Cursor:** правило `.cursor/rules/zvec.mdc` — скопируйте в `<project>/.cursor/rules/`.
- **Прочие агенты** (Claude Code, opencode и т.п.): инструкции в `AGENTS.md`.

> `embed_rubert_worker.py` — опциональный воркер под `sergeyzh/rubert-large-uncased-sts`; по умолчанию не подключается, модель берётся из `ZVEC_EMBEDDING_MODEL`.
